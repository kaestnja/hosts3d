Purpose
    Quick human-readable build notes for local Hosts3D development.
    This file summarizes the expected dependencies, helper scripts, and output layout.

To compile you need the following libraries installed
    libglfw3-dev
    libpcap-dev

Project status
    This repository is an independent continuation of the original Hosts3D 1.15 project.
    Original upstream authorship remains credited to Del Castle.
    See CREDITS.md and the RELEASE_NOTES_*.md files for continuation context.

./configure
make 
make install

Quick script builds:
    ./compile-hosts3d [Release|Debug]
    ./compile-hsen [Release|Debug]
    ./compile-mac-hosts3d [Release|Debug]
    ./package-release-linux [Release|Debug] [arm64|x64|x86]
    compile-hosts3d.bat [Release|Debug] [x86|x64] [--no-pause]
    compile-hsen.bat [Release|Debug] [x86|x64] [--no-pause]

macOS note:
    compile-mac-hosts3d is still kept as a helper script, but the current 1.18 tree
    has not been recently verified on a real macOS machine.
    Treat it as a best-effort path and verify on macOS before relying on it.

Script output layout:
    Release/<os>/<arch>/
    Debug/<os>/<arch>/
    build/windows/<target>/<arch>/<config>/

Build output policy:
    Runtime binaries under Release/ and Debug/ are local build products.
    They should not be kept as tracked Git artifacts.
    Publish distributable binaries as release packages/assets instead.

Parallel Windows build note:
    compile-hosts3d.bat and compile-hsen.bat now use separate object directories per
    target and architecture, so x86/x64 and Hosts3D/hsen builds can run in parallel
    without clobbering each other's object files.

Windows local third-party layout:
    See third_party/README.md

Windows MSYS2 bootstrap:
    Install MSYS2 first from:
        https://www.msys2.org/
    or via:
        winget install -e --id MSYS2.MSYS2
    Then install/update the toolchain packages, for example:
        C:\msys64\usr\bin\bash -lc "pacman -Syu --noconfirm"
        C:\msys64\usr\bin\bash -lc "pacman -S --needed --noconfirm mingw-w64-i686-gcc mingw-w64-i686-binutils mingw-w64-i686-glfw make"
        C:\msys64\usr\bin\bash -lc "pacman -S --needed --noconfirm mingw-w64-x86_64-gcc mingw-w64-x86_64-binutils mingw-w64-x86_64-glfw make"

Windows x64 note:
    x64 builds need an x64 MinGW compiler.
    The GLFW 3 package must be installed in the matching MSYS2 MinGW toolchain root.
    Example packages:
        mingw-w64-i686-glfw
        mingw-w64-x86_64-glfw
    On the current machine, the verified x64 compiler path is:
        C:\msys64\mingw64\bin\g++.exe

Codex note:
    A future Codex session can usually run the pacman/package commands above if MSYS2
    is already installed at C:\msys64.
    The interactive MSYS2 installer itself should still be started by the user.

Windows packaging note:
    package-release-windows.bat defaults to a public ZIP without Packet.dll and wpcap.dll.
    Use:
        package-release-windows.bat Release x64 with-npcap
    when you want a private/local test ZIP that carries the Npcap runtime DLLs.
    The staged release package now keeps its files flat in the package root:
        README.md
        README-runtime-windows.md
        README-testing.md
        Hosts3D.exe / hsen.exe / runtime DLLs
        sim-hsen.* / demo-hsen.*

Linux packaging note:
    package-release-linux creates a tar.gz plus SHA256 file from Release/linux/<arch>.
    For Raspberry Pi 5 the expected arch is usually arm64.
    The staged Linux package uses the same flat layout with README-testing.md and
    the bundled synthetic demo/test scripts beside the runtime binaries.
